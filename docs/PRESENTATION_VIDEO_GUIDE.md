```
"In educational institutions worldwide, attendance management remains 
a significant administrative challenge.

The first issue is TIME CONSUMPTION. Traditional roll-call methods consume 
an average of five to ten minutes per class session. With multiple classes 
daily, this accumulates to hours of lost instructional time each semester.

The second issue is INACCURACY. Paper-based sign-in sheets are vulnerable 
to proxy attendance, where students sign on behalf of absent classmates. 
This compromises the integrity of attendance records.

The third issue is MANUAL DATA HANDLING. Lecturers must manually transfer 
attendance data into digital systems, leading to transcription errors, 
delayed records, and administrative burden.

The fourth issue is LACK OF REAL-TIME VISIBILITY. There is no immediate 
mechanism for lectures to know who is attending, making it difficult to 
track student engagement patterns.

Finally, EXCUSE MANAGEMENT is fragmented. Students submit excuse letters 
through informal channels â€” emails, paper forms, or verbal requests. 
This leads to lost documentation and inconsistent processing.

These problems motivated the development of CObot Plus."
```

### Narration Script
```
"Existing attendance systems in the market have notable limitations.

QR-Code and Bluetooth-based systems, such as those used in some universities, 
require students to actively perform an action â€” scanning a code or enabling 
Bluetooth. This still allows for proxy attendance if a student shares their 
device with a classmate.

Fingerprint-based biometric systems address the proxy problem but create 
bottlenecks. Students must queue to scan their fingerprints one by one, 
consuming significant time, especially in large classes.

Some face recognition systems exist but are often standalone applications 
without integration into a comprehensive class management ecosystem.

CObot Plus differentiates itself in THREE key ways:

First, PASSIVE RECOGNITION. Students simply walk into the classroom. 
No action required. The camera detects and logs attendance automatically.

Second, EDGE COMPUTING. Face recognition runs on a Raspberry Pi device 
at the classroom level, reducing dependency on cloud connectivity and 
enabling real-time processing.

Third, INTEGRATED ECOSYSTEM. CObot Plus is not just an attendance scanner â€” 
it is a complete class management system with scheduling, excuse management, 
analytics, and role-based access control."
```

## SECTION 5: PROPOSED SYSTEM DESIGN

**DURATION: 2 minutes 30 seconds**

### Visual
- System architecture diagram
- Component breakdown
- Technology stack visualization
- Data flow diagram

### Narration Script
```
"Now let me explain the proposed system design of CObot Plus.

The system follows a THREE-TIER ARCHITECTURE.

TIER ONE: The Edge Device Layer.
A Raspberry Pi equipped with a camera module is deployed in each classroom.
The Pi runs a Python Flask server with OpenCV and the face_recognition library.
When students enter, the camera captures video frames, detects faces, 
and matches them against enrolled student face encodings stored in the database.
This edge computing approach enables real-time recognition without 
depending on constant cloud connectivity.

TIER TWO: The Cloud Database Layer.
We use Supabase as our backend-as-a-service platform.
Supabase provides PostgreSQL database, authentication, 
and Row-Level Security policies.
All data â€” student profiles, course information, attendance records, 
and excuse letters â€” is stored securely in the cloud with proper access controls.

TIER THREE: The Web Application Layer.
The frontend is built with Next.js and React, providing a modern, 
responsive user interface.
The application implements ROLE-BASED ACCESS CONTROL with three user types:
Students, Lecturers, and Administrators â€” each with tailored interfaces 
and permissions.

Let me briefly describe each user's experience.

STUDENTS access their personal attendance records through a calendar view, 
showing their attendance status for each class. They can submit excuse 
letters digitally with supporting documents.

LECTURERS see a dashboard with a live camera feed and real-time attendance 
list. They can view analytics including attendance trends, punctuality scores, 
and chronic absentee identification. They also manage excuse letter approvals 
for their assigned sections.

ADMINISTRATORS have full system access to manage students, courses, sections, 
schedules, and user roles.

The system operates in AUTOMATIC MODE based on class schedules.
Attendance windows are defined per schedule â€” for example, the first fifteen 
minutes for 'Present' status, the next fifteen minutes for 'Late' status.
No manual start or end of class is required."
```
